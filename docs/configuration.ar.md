# إعدادات

تكوين طبقات Solar AI Optimizer من عدة مصادر. لوحة **الإعدادات** في
لوحة القيادة هي الواجهة الأساسية للمشغلين.

## مصادر التكوين

| المصدر | الغرض |
|--------|---------|
| **واجهة المستخدم للإعدادات** | التكوين الأساسي: اتصال HA، المنطقة الزمنية للموقع، خريطة الكيان، البطارية، الاحتياطي، التنبؤ، التحكم |
| `config/config.yaml`(بالصورة) | الإعدادات الافتراضية الأساسية؛ تجاوزات واجهة المستخدم المخزنة في حجم البيانات |
| `config.runtime.yaml`(حجم البيانات) | تعديلات واجهة المستخدم المستمرة (مدمجة بعمق في القاعدة) |
| `.env`/ يؤلف`environment`| الأسرار، وعلامات الميزات، ورمز واجهة برمجة التطبيقات الاختياري |
| HA الوظيفة الإضافية`options.json`| تم تعيينه إلى env vars بواسطة`run.sh`عند التشغيل كوظيفة إضافية |

الأقسام الرئيسية (جميعها قابلة للتحرير في الإعدادات):

- اتصال مساعد المنزل
- **الموقع** — المنطقة الزمنية لـ IANA أو **تلقائي** (من Open-Meteo عند خط العرض/خط الطول للموقع)؛ إحداثيات الموقع يحرك حدود يوم التنبؤ، ومجموعات التعلم الخاصة بالحمل/درجة الحرارة، والطوابع الزمنية للوحة المعلومات
- خريطة الكيان العاكس (أجهزة استشعار القراءة + عناصر التحكم في الكتابة)
- مواصفات البطارية وسياسة الاحتياطي
- مزود التنبؤ، المصفوفات الكهروضوئية، نموذج درجة الحرارة
- التحكم في التوقيت ووضع المحرك (`rules`أو`mpc`)
- **تمكين النظام الفرعي** —`engine.enabled`(احتياطي/MPC/توقعات)،`grid_charge.enabled`(يكتب العاكس شبكة الشحن)،`load_shedding.enabled`(مفاتيح الطبقة)؛ يمكن تبديل كل منها بشكل مستقل في الإعدادات أو علامة تبويب فصل الأحمال
- نبضات قلب آمنة من الفشل

تم تكوين **فصل الأحمال** في علامة التبويب المخصصة **فصل الأحمال** (وليس الإعدادات).
يرى[دليل مستخدم لوحة المعلومات ← طبقات فصل الأحمال](frontend-manual.md#load-shedding-tiers).

يرى[إعداد مساعد المنزل](home-assistant-setup.md)للاتصال ورسم خرائط الكيان،
و[دليل مستخدم لوحة المعلومات → الإعدادات](frontend-manual.md#settings)للحصول على إرشادات تفصيلية لواجهة المستخدم.

## متغيرات البيئة الاختيارية

موثق في [`.env.example`](https://github.com/oraad/solar-ai-optimizer/blob/main/.env.example).
التجاوزات الشائعة:

| متغير | الغرض |
|----------|---------|
| `HA_BASE_URL` / `HA_TOKEN`| اتصال مساعد المنزل (أو تم تعيينه في واجهة المستخدم) |
| `SHADOW_MODE` | `true`= لاحظ فقط (افتراضي) |
| `LOCAL_ADMIN_PASSWORD_HASH` / `SESSION_SECRET`| تسجيل دخول المسؤول المحلي للوصول المستقل |
| `TRUST_INGRESS_HEADERS`| الثقة بدخول HA إلى رؤوس المستخدم والسماح بإطار iframe للشريط الجانبي (`SAMEORIGIN`; تلقائي عند الوظيفة الإضافية) |
| `API_TOKEN`| الرمز المميز للنصوص البرمجية؛ يحمي API عند التعيين |
| `CORS_ORIGINS`| أصول CORS مفصولة بفواصل (افتراضي`*`) |
| `ML_LOAD_ENABLED`| توقعات الحمل المعزز للتدرج (يحتاج إلى sklearn في الصورة) |
| `LLM_ENABLED` / `OLLAMA_*`| مساعد LLM محلي (المرحلة 4) |
| `DEMO_MODE`| **المستندات فقط** — القياس الاصطناعي عن بعد؛ أبدا في الإنتاج |

متى`API_TOKEN`تم ضبطه، أدخل نفس القيمة في **الإعدادات → أمان واجهة برمجة التطبيقات**.

## Solcast (مزود طاقة شمسية اختياري)

في الإعدادات، اضبط **التوقعات → الموفر** على`solcast`. لا يتم تخزين بيانات الاعتماد في
تكوين واجهة المستخدم - تعيين متغيرات البيئة (أو خيارات الوظائف الإضافية HA):

| متغير | الغرض |
|----------|---------|
| `SOLCAST_API_KEY`| رمز حامل من حساب Solcast الخاص بك |
| `SOLCAST_RESOURCE_ID`| معرف موقع السطح من لوحة معلومات Solcast |

يجب تعيين كلاهما عندما يكون الموفر`solcast`; وإلا فسيعود التطبيق إلى Open-Meteo
ويظهر تحذيرًا بشأن التكوين الخاطئ.

## التسجيل

| متغير | القيم | الافتراضي |
|----------|--------|---------|
| `LOG_LEVEL`| تصحيح، معلومات، تحذير، خطأ | معلومات |
| `LOG_FORMAT` | `text`, `json`| نص |

تعيين`LOG_FORMAT=json`لمجمعات سجل الإنتاج (كائن JSON واحد لكل سطر).

## أوضاع المحرك

| الوضع | الوصف |
|------|-------------|
| `rules`| محرك القاعدة الافتراضي — الاحتياطي الديناميكي، واستخدام الشبكة التفاعلية |
| `mpc`| إرسال بطارية LP اختيارية (يتطلب PuLP في الصورة) |

اضبط **وضع المحرك →** في الإعدادات. تعود MPC إلى القواعد في حالة عدم توفر PuLP.

### ترتيب أولويات التحسين

في **الإعدادات → المحرك**، أعد الترتيب`priority_order`(الافتراضي: المرونة، والادخار،
الاكتفاء الذاتي). يجب أن تتضمن القائمة كل قيمة مرة واحدة بالضبط. مثال في
`config.runtime.yaml`:

```yaml
engine:
  mode: rules
  priority_order:
    - resilience
    - savings
    - self_sufficiency
```

تؤثر الأولويات الأعلى مرتبة على الاحتياطيات الاحتياطية، وتسجيل مخاطر انقطاع التيار الكهربائي، وMPC
الأوزان الموضوعية وقوة عامل منحدر شحن الشبكة. **الادخار** يعني
الاستخدام الانتهازي للشبكة عندما يكون موجودًا - وليس وقت الاستخدام أو تحسين التعريفة.

## سفك الأحمال

قم بتكوين الطبقات في علامة التبويب لوحة المعلومات **فصل الأحمال**. يمكن التحكم في كل طبقة
العديد من مفاتيح الطاقة (`switch.*`أو`input_boolean.*`) التي تسلط وتستعيد
معًا باستخدام تباطؤ SOC. **الأولوية** الدنيا تتساقط أولاً.

الكيانات المصاحبة على نفس جهاز Home Assistant (المناخ، التحديد، المروحة، وما إلى ذلك)
يتم اكتشافها تلقائيًا والتقاطها عند تساقطها؛ يتم استعادتها عندما
الطبقة تعود. الأجهزة التي ** تم إيقاف تشغيلها قبل تساقطها ** لا يتم تشغيلها أبدًا
عن طريق الاستعادة.

يتم التقاط اللقطات **مرة واحدة لكل حلقة فصل أحمال** (أول مرة يُفصل فيها كيان
الطاقة) وتُحفظ حتى الاستعادة أو المسح أو تقليم الإعدادات. دورات الفصل اللاحقة
بينما يبقى المفتاح مطفأً لا تعيد الالتقاط ولا تستبدل تلك اللقطة. قد تُلتقط اللقطة
حتى عندما يكون مراقب كتابة HA قديمًا؛ تبقى كتابة الإيقاف حتى يصبح HA حديثًا.

| المجال | الغرض |
|-------|---------|
| `restore_enabled`| استعادة على SOC متى`soc >= restore_above_soc` |
| `restore_on_grid`| استعادة عند وجود الشبكة (إذا كانت العلامة العامة قيد التشغيل) |
| `state_entities`| خريطة اختيارية لكيان الطاقة → معرفات الكيانات المصاحبة |

حذف مفتاح في`state_entities`لاكتشاف الرفاق تلقائيًا؛ تعيين`[]`ل
سفك التبديل فقط. تستمر اللقطات ضمن حجم البيانات ويتم تقليمها
عندما يتغير تكوين الطبقة.

واجهة برمجة التطبيقات (المسؤول):`GET /api/shed/device-companions?entity_id=…`معاينات الاكتشاف؛
`GET /api/shed/snapshots`يسرد حالة ما قبل السقيفة المخزنة.

يرى[إعداد المساعد المنزلي → فصل الأحمال](home-assistant-setup.md#load-shedding)
للحصول على أمثلة الكيان.

## إضافات بناء عامل الميناء

تقوم الصورة الافتراضية بتثبيت إضافات المرحلة 3/4 (PuLP وscikit-learn وnumpy) عبر`INSTALL_EXTRAS=1`.
للحصول على صورة هزيلة:

```bash
docker compose build --build-arg INSTALL_EXTRAS=0
```

## النسخ الاحتياطي للبيانات

ال`solar-data`حجم عامل الإرساء (أو الوظيفة الإضافية`/data`) يحمل:

- `solar.db`— القياس عن بعد وتاريخ التدقيق
- `config.runtime.yaml`- تجاوزات تكوين واجهة المستخدم
- `model.json`— تعلمت تحيز التنبؤ / ملف تعريف التحميل

قم بعمل نسخة احتياطية لوحدة التخزين هذه قبل الترقيات. يرى[نشر Proxmox → النسخ الاحتياطي](proxmox.md#backup)
على سبيل المثال أمر القطران.

## الترقية

```bash
docker compose up -d --build
```

تتطلب نقاط نهاية واجهة برمجة التطبيقات (API) المتغيرة`Authorization: Bearer <token>`متى`API_TOKEN`تم تعيينه.
