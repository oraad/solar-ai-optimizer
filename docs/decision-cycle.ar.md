# كيف يقرر دورة التحكم

كل حلقة هي **استشعار → قرار → تنفيذ → تحقق**.

1. **المدخلات** — القياس عن بُعد، التوقعات، إحصاءات الشبكة (ملخص يُخزَّن مع القرار).
2. **الاحتياطي** — تحسب القواعد جسر الطاقة الشمسية مقابل أرضية الاستقلالية؛ قد يستبدل MPC أو تثبيت المشغّل الهدف (`source`: `rules` | `mpc` | `operator`). يستخدم الخطر الهدف **الفعّال**.
3. **شحن الشبكة** — سلسلة سقوف التفاعل/المنحدر تحدد التفعيل والأمبير؛ العامل الملزم هو أدنى سقف.
4. **فصل الأحمال** — سياسة SOC/الشبكة؛ الاستعادة فقط للكيانات ذات لقطة فصل.
5. **التنفيذ** — كتابات HA خارج وضع الظل وخارج إيقاف الكتابة (`paused_grid_charge` / `paused_shedding`). يستمر التخطيط عندما يكون التحسين «متوقفًا».
6. **التحقق** — تعرض النظرة العامة المقصود مقابل المطبَّق للاحتياطي وشحن الشبكة؛ اربط السجل عبر `cycle_id`.

إذا اختلفت النظرة العامة عن السلوك المتوقع، استخدم **Live forensics** (مسؤول) أو MCP `solar_explain_decision` مع قسم `causality` — انظر [mcp.md](mcp.md).
