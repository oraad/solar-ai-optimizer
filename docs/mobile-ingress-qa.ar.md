# دخول الهاتف المحمول QA (رفيق مساعد المنزل)

استخدم قائمة التحقق هذه عند التحقق من صحة لوحة المعلومات داخل تطبيق HA Companion لإدخال iframe (الوظيفة الإضافية الأصلية أو hass_ingress).

## يثبت

1. افتح **Solar AI** من الشريط الجانبي HA على الهاتف (تطبيق iOS و/أو Android Companion).
2. اختبر حسابي **admin** و**viewer** إن أمكن.

## التخطيط والتمرير

- [ ] لا يوجد شريط تمرير عمودي مزدوج (يتم تمرير محتوى التطبيق HA chrome + معًا بشكل طبيعي).
- [ ] يظل الشريط العلوي لزجًا؛ المحتوى لا يجلس تحت درجة الجهاز.
- [ ] يقوم المحتوى السفلي والخبز المحمص بمسح مؤشر الصفحة الرئيسية / الشريط السفلي HA.

## ملاحة

- [ ] يتم تمرير علامات التبويب الرئيسية أفقيًا عند الحاجة؛ تظل التسميات مرئية (وليست أيقونة فقط).
- [ ] من السهل النقر على أزرار علامات التبويب (يصل طولها إلى 44 بكسل تقريبًا).
- [ ] يتم تمرير علامات التبويب الفرعية للسجل (المخطط، والقرارات، وأحداث الشبكة، ...) أفقيًا على الشاشات الضيقة.

## شريط الحالة (عرض ≥600 بكسل)

- [ ] الحبوب الأولية مرئية: HA، مباشر/ظل، متوقف مؤقتًا (عندما يكون نشطًا)، عارض (عند الاقتضاء).
- [ ] يتم طي التنبيهات الثانوية في قائمة **N تنبيهات**؛ تفتح القائمة وتسرد العناصر.
- [ ] يظل تبديل السمة قابلاً للوصول.

## لوحات المحتوى

- [ ] بطاقات النظرة العامة مكدسة في عمود واحد.
- [ ] يتم عرض مخطط التاريخ بارتفاع منخفض؛ أسطورة قابلة للقراءة.
- [ ] يتم تمرير جداول السجل أفقيًا دون كسر تخطيط الصفحة.
- [ ] نصائح المعلومات (ⓘ) تفتح عند النقر وتغلق عند النقر الخارجي أو Escape.
- [ ] يمكن النقر فوق عناصر التحكم المجزأة.
- [ ] يبقى إدخال الدردشة المساعد أعلى لوحة المفاتيح / المنطقة الآمنة.

## الانحدار (سطح المكتب)

- [ ] مستقل`http://localhost:8000`ولا يزال إدخال سطح المكتب في الشريط الجانبي HA يبدو صحيحًا عند 1280 × 900.

## لقطات الشاشة التلقائية

مع تشغيل المكدس التجريبي:

```bash
docker compose --profile docs run --rm docs-screenshots
```

أو محليا من`frontend/`بعد مرة واحدة`npm ci`و`npx playwright install chromium`:

```bash
npm run docs:screenshots
```
